# Project Manifest — Гороскоп

## Стек
React 18 + TypeScript, Vite, Tailwind CSS v4, Lucide React, Manrope

## Проект
PWA-приложение (MAX Mini Apps) с ежедневными астрологическими рекомендациями на основе математических алгоритмов (лунные фазы, стихии, знаки зодиака). Три экрана: Ритм, Любовь, Энергия.

## ЦА
Поколение Z и молодые взрослые (18–30 лет), интересующиеся self-care, астрологией и персональными рекомендациями. Россияне, пользователи MAX мессенджер.

## Дизайн-стиль: Quiet Luxury («Тихая роскошь»)

**Quiet Luxury** (тихая роскошь) — философия дизайна, где ценность передаётся через качество, а не через декор. Минимум логотипов, максимум внимания к материалу, пропорциям и свету.

### Как это выглядит в проекте:
- **Цветовая палитра**: глубокий тёмный фон `#050510`, приглушённые акценты (pink, violet, indigo) без кричащих оттенков
- **Материалы**: стеклянные карточки (`backdrop-blur-2xl`), тонкие полупрозрачные границы (`border-white/10`), матовый эффект
- **Типографика**: Manrope, черное начертание, крупные кегли, минимум families
- **Пространство**: много воздуха, отсутствие лишних линий и рамок, контент «дышит»
- **Анимация**: плавная, незаметная — fade-in, slide-in, длительность 300–700ms
- **Отсутствие**: градиентных заливок, массивных теней, бейджей, лишних иконок

### Quiet Luxury — это не про:
- Золото и блёстки (это лакшери)
- Массивные рамки и тяжёлые тени
- Логотипы и брендинг на каждом углу
- Крикливый маркетинг (скидки, акции)

## Философия разработки: подход Apple / Стив Джобс

Мы строим продукт по тем же принципам, что Apple:

1. **Simplicity is the ultimate sophistication** — каждая функция существует, только если она делает продукт понятнее. Всё лишнее отсекается.
2. **End-to-end контроль** — дизайн и код растут из одной головы. Нет разрыва между макетом и реализацией.
3. **Собственные алгоритмы** — никаких внешних API, вся астрология считана внутри. Как Apple делает свои чипы, мы делаем свои движки.
4. **Детали решают всё** — скругления, отступы, тайминги анимаций. Каждый пиксель выверен.
5. **Меньше — лучше, чем больше** — экран не перегружен. Пользователь должен найти ответ за один взгляд.
6. **Работает как ожидается** — интерфейс не требует инструкций. Кнопка выглядит как кнопка, свайп работает как свайп.
